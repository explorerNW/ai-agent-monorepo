### 🛠️ 严格工作流（请严格按顺序执行）

1. **组件层级分析与数据建模**：
   - 调用 MCP 工具读取设计稿的布局、间距、颜色 Token 和字体排版数据。
   - **识别原子组件**：分析设计稿，将可复用的单元（按钮、卡片、头像、标签等）拆分为独立组件。
   - **数据驱动思维**：严禁写死业务文案！分析组件中哪些文本、状态是动态数据，并在 `interface` 中为其定义清晰的 Props 类型。

2. **架构级图片性能优化策略（最高优先级 - 绝对执行）**：
   - **禁止内联 SVG**：**绝对禁止**将位图、照片或复杂插画生成内联 SVG、`<pattern>` 或复杂的矢量路径！
   - **强制占位符规则**：遇到任何位图节点，**必须**生成标准的 HTML `<img>` 标签作为占位符。
   - **差异化加载策略**：
     - **首屏关键图片**（如 Banner、Hero 区域、首屏大图）：必须添加 `loading="eager"` 和 `decoding="async"`。
     - **非首屏图片**（如列表图、底部插图）：必须添加 `loading="lazy"` 和 `decoding="async"`。
   - **防布局偏移（CLS）**：所有 `<img>` 标签必须保留设计稿要求的 `width` 和 `height`（或通过 CSS/Tailwind 设置明确宽高）。
   - **占位符样式规范**：`src` 属性统一设置为 `"#"`，必须添加 `alt="图片占位符"` 和 `className="bg-gray-200 object-cover"`。
   - **图标节点**：只有纯粹的几何图形、单色 UI 图标，才允许匹配 `lucide-react` 等主流图标库。

3. **React 代码级性能优化（核心）**：
   - **防止无效重渲染**：
     - 对所有纯展示型的原子子组件（如 Card, Button, Badge 等），**必须使用 `React.memo` 进行包裹**。
     - 对于组件内部传递给子组件或 `useEffect` 的函数，**必须使用 `useCallback` 缓存引用**。
     - 对于组件内部复杂的计算逻辑（如大数据过滤、排序），**必须使用 `useMemo` 缓存计算结果**。
   - **列表渲染优化**：
     - 渲染列表时，**绝对禁止使用 `index` 作为 `key`**，必须使用数据中唯一的 `id`。
     - 如果设计稿包含超长列表（预估超过 50 条），请提示我是否需要引入 `react-window` 实现虚拟滚动。
   - **代码分割**：
     - 对于非首屏的重型组件（如 Modal, Drawer, Chart），请使用 `React.lazy` 和 `Suspense` 进行动态导入。

4. **代码注释与文档规范（新增）**：
   - **组件级文档（JSDoc/TSDoc）**：每个组件的顶部**必须**包含标准的 JSDoc/TSDoc 注释块，清晰说明组件的**功能用途**、核心 Props 的**业务含义**以及特殊的使用注意事项。
   - **关键逻辑注释**：在复杂的业务逻辑、非常规的算法、或者为了解决特定 Bug/兼容性问题而写的代码块上方，**必须**添加单行或多行注释，重点解释**“为什么这么写（Why）”**以及背后的业务考量。
   - **拒绝无效注释**：**严禁**为显而易见的代码（如简单的变量赋值、直观的 JSX 结构、常规的 React Hooks 调用）添加“做什么（What）”的废话注释。保持代码的清爽与高信噪比。

5. **设计规整与容错（智能还原）**：
   - **视觉规整**：如果设计稿中存在微小的像素偏差（如 15px 和 16px 的间距混用），请自动规整为统一的 4px/8px 网格系统或 Tailwind 预设值。
   - **响应式与弹性布局**：使用 `w-full`, `flex-1`, `md:flex-row` 等 Tailwind 类，严禁写死绝对宽度。
   - **无障碍性（a11y）**：所有交互元素必须具备键盘可访问性，图标按钮必须添加 `aria-label`，表单控件必须使用 `<label htmlFor="...">` 关联。

6. **文件目录与路径规划**：
   - 主页面/父组件命名为 `PageSection.tsx`。
   - 拆分出的子组件必须放置在 `components/` 目录下（例如：`components/UserCard.tsx`）。
   - 在主组件中，使用相对路径正确引入子组件。
